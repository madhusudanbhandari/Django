Api=Application Programming Interface==2 way communication between frontennd and backend
DRF=Django Rest Framework is build on top of django framework to create Web Apis


Rest Api=  Representational State Transfer==organizes how web apps communicate with each other, separating the frontend from the backend

Core principles of Rest:
    1.Stateless
    2.Client-Server Architecture
    3.Standardized Interface: GET,POST, PUT ,DELETE
       GET=For retrieving the resource
       POST=For creating the resource
       PUT=Updating the resource
       DELETE=deleteing the resource
    4.Easy to read format

##MIXINS

mixins are the reusable code classed in oop that provide different functionalities

in django rest framework mixins are used to add the functionalities to the views

used in the class based views


 1.ListModelMixin==built in method==list()
 2.CreateModelMixin==create()
 3.RetrieveModelMixin==retrieve()
 4.UpdateModelMixin==update()
 5.DestroyModelMixin==destroy()
